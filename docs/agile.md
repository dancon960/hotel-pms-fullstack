# Investigación: Metodologías de Desarrollo

### 1. Metodología Agile
La metodología Agile es un enfoque de desarrollo de software que prioriza la flexibilidad y las entregas rápidas. En lugar de planificar todo el proyecto de principio a fin de forma rígida, se trabaja en ciclos cortos para poder corregir errores o añadir cambios sobre la marcha. El objetivo principal es aportar valor al usuario desde las primeras etapas del desarrollo.

### 2. Scrum
Scrum es un marco de trabajo dentro de Agile que se basa en la disciplina y la organización por tiempos. Sus conceptos clave son:

* **Sprints:** Ciclos de trabajo fijos (normalmente de 2 a 4 semanas) donde se debe terminar una serie de tareas.
* **Roles:** Existen figuras como el Scrum Master (facilitador) y el Product Owner (responsable de la visión del producto).
* **Eventos:** Se realizan reuniones diarias cortas para revisar el progreso y planificar el día.

Es un sistema ideal para proyectos que necesitan una estructura muy marcada.

### 3. Kanban
Kanban es un método visual para gestionar tareas. A diferencia de Scrum, no se basa en tiempos fijos, sino en un flujo continuo de trabajo. Se utiliza un tablero con columnas (To Do, Doing, Done) para visualizar el estado de cada tarea. El objetivo es evitar los cuellos de botella, limitando la cantidad de trabajo que puede haber en la columna de "En proceso".

### 4. Comparativa y uso
* **Diferencia principal:** Scrum es más rígido y se basa en calendarios cerrados (Sprints), mientras que Kanban es más fluido y se adapta al ritmo de salida de las tareas.
* **Cuándo usar cada una:**
    * **Scrum:** Recomendable para proyectos grandes donde el equipo necesita una coordinación constante y entregas periódicas obligatorias.
    * **Kanban:** Ideal para equipos más pequeños o para tareas donde el trabajo llega de forma constante y se necesita flexibilidad total.
