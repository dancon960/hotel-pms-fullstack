# Organización del Proyecto

Para el desarrollo de esta aplicación, se ha optado por una gestión visual basada en **Kanban**. 

### Herramienta de gestión
He utilizado **Trello** como centro de organización. El tablero se ha estructurado en las siguientes columnas:
* **Backlog:** Ideas y funcionalidades pendientes de definir técnicamente.
* **To Do:** Tareas listas para empezar a programar.
* **In Progress:** Tareas en las que estoy trabajando actualmente.
* **Done:** Funcionalidades terminadas y probadas.

### Metodología de trabajo
He dividido el proyecto en hitos técnicos. Primero, la configuración del entorno y arquitectura de carpetas; segundo, la definición de tipos y datos de ejemplo; y tercero, la creación de componentes y lógica de estados. Este enfoque me ha permitido mantener el código limpio y evitar errores de integración.
